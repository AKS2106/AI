Patient Outcomes & Clinical Analytics Pipeline

A HIPAA-compliant, medallion-architecture data pipeline on AWS — ingesting EHR, claims, lab, and pharmacy data into a clinical analytics warehouse.

Business Problem

MedCore Health Systems is a regional hospital network with 12 hospitals, 80 outpatient clinics, and 2.4 million active patients. Clinical data is trapped in 5 separate systems — Epic EHR, insurance claims processors, lab information systems, pharmacy dispensing systems, and patient satisfaction surveys. The analytics team cannot answer critical questions like "Which diabetic patients are at high risk of 30-day readmission?" or "What is our average time-to-treatment by department?"

Why it matters

  • CMS penalizes hospitals up to 3% of Medicare reimbursements for excess 30-day readmissions — $8M/year exposure for MedCore
  • No unified patient timeline across EHR, pharmacy, and lab — clinicians make decisions with incomplete history
  • Reporting to state health departments takes 3 weeks of manual data pulls
  • No ability to identify care gaps (patients overdue for screenings, not filling prescriptions)
  • Population health management programs cannot be measured for ROI

HIPAA Compliance — What Every DE Must Know

HIPAA (Health Insurance Portability and Accountability Act) requires safeguards for Protected Health Information (PHI). As a data engineer, you are a "Business Associate" — violations can result in fines up to $1.9M per violation category per year.

The 18 PHI identifiers you must protect

Any dataset containing any of these must be treated as PHI:

1. Patient names
2. Geographic data (zip code)
3. Dates (except year)
4. Phone numbers
5. Email addresses
6. Social Security numbers
7. Medical record numbers
8. Health plan numbers
9. Account numbers
10. Certificate/license numbers
11. Device identifiers
12. Web URLs
13. IP addresses
14. Biometric identifiers
15. Full-face photographs
16. Vehicle identifiers
17. Fax numbers
18. Any unique identifying number

Technical safeguards checklist (your pipeline must implement all)

  • Encryption at rest: S3 SSE-KMS with customer-managed key (CMK)
  • Encryption in transit: TLS 1.2+ for all data movement (S3, Redshift, Glue)
  • Access control: IAM roles with least privilege — no human user has direct S3 access to PHI buckets
  • Audit logging: CloudTrail for all API calls, S3 server access logs, Redshift audit logs
  • De-identification in Silver layer: replace direct identifiers with surrogate keys before analytics use
  • Data retention policy: S3 Lifecycle rules to move/delete data per retention schedule
  • VPC isolation: Redshift and Glue run inside a VPC with no internet gateway

De-Identification Module

# src/transformation/deidentification.py
import hashlib, hmac, os
from pyspark.sql import DataFrame
from pyspark.sql.functions import col, udf, lit, sha2, concat
from pyspark.sql.types import StringType

# HMAC-based pseudonymization — reversible by authorized parties
# with the secret key, NOT reversible without it
DEIDENT_SECRET = os.environ.get("DEIDENT_SECRET")  # stored in AWS Secrets Manager

def pseudonymize_id(value: str) -> str:
    """Replace direct patient ID with HMAC-SHA256 surrogate key."""
    if value is None: return None
    h = hmac.new(DEIDENT_SECRET.encode(), value.encode(), hashlib.sha256)
    return h.hexdigest()[:32]

pseudonymize_udf = udf(pseudonymize_id, StringType())

def deidentify_encounters(df: DataFrame) -> DataFrame:
    """
    Apply HIPAA Safe Harbor de-identification to encounters DataFrame.
    Removes all 18 PHI identifiers, retains clinical/analytical fields.
    """
    return (df
        # Pseudonymize identifiers (keeps linkability for longitudinal analysis)
        .withColumn("patient_id",  pseudonymize_udf(col("patient_id")))
        .withColumn("provider_id", pseudonymize_udf(col("attending_provider_id")))

        # Generalize dates — keep year+month only (remove exact day for DOB)
        .withColumn("admit_year",   col("admit_datetime").cast("date").cast("string")[:7])
        .withColumn("admit_date",   col("admit_datetime").cast("date"))
        .withColumn("birth_year",   col("dob").cast("date").cast("string")[:4].cast("int"))

        # Drop all direct PHI columns
        .drop("patient_name", "dob", "admit_datetime",
              "discharge_datetime", "attending_provider_id")

        # Generalize geography — truncate zip to 3 digits
        # (if zip count < 20K residents, replace with 000 per Safe Harbor)
        .withColumn("deident_flag", lit(True))
    )

Bronze → Silver Glue Job

# src/transformation/bronze_to_silver.py
import sys, logging
from awsglue.utils import getResolvedOptions
from pyspark.context import SparkContext
from awsglue.context import GlueContext
from pyspark.sql import functions as F
from pyspark.sql.types import *
from deidentification import deidentify_encounters
from data_quality import DataQualityChecker

logger = logging.getLogger("bronze_to_silver")
args   = getResolvedOptions(sys.argv, ["JOB_NAME", "process_date"])
sc     = SparkContext()
glue   = GlueContext(sc)
spark  = glue.spark_session

BUCKET    = "medcore-phi-data-lake"
PROC_DT   = args["process_date"]
y, m, d   = PROC_DT.split("-")

## ── READ BRONZE (JSON) ────────────────────────────
bronze_path = f"s3://{BUCKET}/bronze/encounters/year={y}/month={m}/day={d}/"

df_raw = (spark.read
    .option("multiLine", True)
    .json(bronze_path))

logger.info(f"Bronze records: {df_raw.count()}")

## ── SCHEMA VALIDATION ─────────────────────────────
REQUIRED_COLS = ["encounter_id", "patient_id", "admit_datetime",
                 "primary_icd10", "department"]
missing = [c for c in REQUIRED_COLS if c not in df_raw.columns]
if missing:
    raise ValueError(f"Schema violation — missing columns: {missing}")

## ── DATA QUALITY ──────────────────────────────────
DataQualityChecker(df_raw, "bronze_encounters") \
    .check_nulls(["encounter_id", "patient_id", "primary_icd10"]) \
    .check_duplicates("encounter_id") \
    .check_range("length_of_stay_days", 0, 365) \
    .summary()

## ── TYPE CASTING ──────────────────────────────────
df_typed = (df_raw
    .withColumn("admit_datetime",     F.to_timestamp("admit_datetime"))
    .withColumn("discharge_datetime", F.to_timestamp("discharge_datetime"))
    .withColumn("readmission_flag",   col("readmission_flag").cast("boolean"))
    .withColumn("length_of_stay_days",col("length_of_stay_days").cast("int"))
    .withColumn("icd10_code", F.upper(F.trim(col("primary_icd10"))))
)

## ── DE-IDENTIFY (HIPAA Safe Harbor) ───────────────
df_deident = deidentify_encounters(df_typed)

## ── STANDARDIZE ICD-10 CODES ──────────────────────
# Join with reference table to add readable diagnosis category
icd_ref = spark.read.option("header", True).csv(
    f"s3://{BUCKET}/reference/icd10_codes/icd10.csv"
)
df_silver = (df_deident
    .join(F.broadcast(icd_ref), df_deident.icd10_code == icd_ref.code, "left")
    .withColumn("year",  F.lit(y))
    .withColumn("month", F.lit(m))
    .withColumn("day",   F.lit(d))
    .withColumn("pipeline_version", F.lit("1.0"))
    .withColumn("processed_at", F.current_timestamp())
)

## ── WRITE SILVER (Parquet) ─────────────────────────
silver_path = f"s3://{BUCKET}/silver/encounters/"
(df_silver.write
    .mode("overwrite")
    .partitionBy("year", "month", "day")
    .parquet(silver_path))

logger.info(f"✅ Silver written: {df_silver.count()} records")

Silver → Gold: Readmission Risk Scoring

# src/transformation/silver_to_gold.py
from pyspark.sql import functions as F
from pyspark.sql.window import Window

## Read silver encounters
silver = spark.read.parquet(f"s3://{BUCKET}/silver/encounters/")
silver_rx = spark.read.parquet(f"s3://{BUCKET}/silver/pharmacy/")
silver_lab = spark.read.parquet(f"s3://{BUCKET}/silver/lab_results/")

## ── PATIENT RISK SCORE (rule-based) ───────────────
w_patient = Window.partitionBy("patient_id")

patient_features = (silver
    .groupBy("patient_id")
    .agg(
        F.count("encounter_id").alias("total_encounters"),
        F.sum(F.col("readmission_flag").cast("int")).alias("total_readmissions"),
        F.avg("length_of_stay_days").alias("avg_los_days"),
        F.max("admit_date").alias("last_encounter_date"),
        F.countDistinct("icd10_code").alias("distinct_conditions"),
        F.max(F.when(F.col("insurance_type") == "Medicare", 1).otherwise(0))
            .alias("is_medicare")
    )
)

# Count chronic meds (proxy for chronic condition burden)
rx_burden = (silver_rx
    .groupBy("patient_id")
    .agg(F.countDistinct("medication_name").alias("distinct_meds"))
)

# Join features and compute composite risk score
risk_scores = (patient_features
    .join(rx_burden, "patient_id", "left")
    .withColumn("distinct_meds", F.coalesce("distinct_meds", F.lit(0)))
    .withColumn("readmission_rate",
        F.round(F.col("total_readmissions") / F.col("total_encounters"), 3))
    .withColumn("risk_score",
        F.round(
            (F.col("total_readmissions") * 3.0) +
            (F.col("avg_los_days") * 0.5) +
            (F.col("distinct_conditions") * 1.5) +
            (F.col("distinct_meds") * 0.8) +
            (F.col("is_medicare") * 2.0),
        2))
    .withColumn("risk_tier",
        F.when(F.col("risk_score") >= 15, "HIGH")
         .when(F.col("risk_score") >= 8,  "MEDIUM")
         .otherwise("LOW"))
    .withColumn("scored_at", F.current_timestamp())
)

# Write to gold layer
(risk_scores.write
    .mode("overwrite")
    .parquet(f"s3://{BUCKET}/gold/patient_risk_scores/"))

logger.info("✅ Gold risk scores written")

Key schema design decisions

  • dim_patient uses SCD Type 2 — insurance changes affect cost analysis; history matters for regulatory audits
  • dim_diagnosis is a static reference to ICD-10 — never a snowflake from the fact table; avoids repeated string joins
  • patient_risk_scores is a separate derived table — scoring logic may change, keeping it separate protects the core schema
  • No direct PHI in any warehouse table — all identifiers are pseudonymized surrogate keys
  • fact_encounters DISTKEY on patient_sk — most joins are patient-centric

Clinical Analytics SQL Queries

1. 30-day readmission rate by department and month

WITH encounters_ranked AS (
    SELECT
        e.patient_sk,
        e.admit_date,
        e.discharge_date,
        e.department,
        e.readmission_flag,
        d.year,
        d.month,
        LEAD(e.admit_date) OVER (
            PARTITION BY e.patient_sk
            ORDER BY e.admit_date
        ) AS next_admit_date
    FROM fact_encounters e
    JOIN dim_date d ON e.date_sk = d.date_sk
    WHERE e.discharge_date IS NOT NULL
),
readmit_30 AS (
    SELECT *,
        CASE WHEN
            DATEDIFF(day, discharge_date, next_admit_date) <= 30
            THEN 1 ELSE 0
        END AS readmit_30_flag
    FROM encounters_ranked
)
SELECT
    department,
    year,
    month,
    COUNT(*) AS total_discharges,
    SUM(readmit_30_flag) AS readmissions_30d,
    ROUND(AVG(readmit_30_flag::FLOAT) * 100, 2) AS readmission_rate_pct,
    AVG(AVG(readmit_30_flag::FLOAT) * 100) OVER (
        PARTITION BY department
        ORDER BY year, month
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) AS rolling_3mo_rate_pct
FROM readmit_30
GROUP BY 1,2,3
ORDER BY department, year, month;

2. High-risk patient care gap identification

WITH diabetic_patients AS (
    SELECT DISTINCT e.patient_sk
    FROM fact_encounters e
    JOIN dim_diagnosis dd ON e.diagnosis_sk = dd.diagnosis_sk
    WHERE dd.icd10_code LIKE 'E11%'  -- Type 2 diabetes codes
),
last_hba1c AS (
    SELECT
        l.patient_sk,
        MAX(l.result_date) AS last_hba1c_date,
        MAX(l.result_value) AS last_hba1c_value
    FROM fact_lab_results l
    WHERE l.test_name = 'HbA1c'
    GROUP BY 1
),
last_rx AS (
    SELECT
        rx.patient_sk,
        MAX(rx.dispensed_date) AS last_rx_date
    FROM fact_pharmacy rx
    WHERE rx.medication_name ILIKE '%metformin%'
    GROUP BY 1
)
SELECT
    dp.patient_sk,
    rs.risk_tier,
    rs.risk_score,
    h.last_hba1c_date,
    h.last_hba1c_value,
    DATEDIFF(day, h.last_hba1c_date, CURRENT_DATE) AS days_since_hba1c,
    rx.last_rx_date,
    DATEDIFF(day, rx.last_rx_date, CURRENT_DATE)  AS days_since_metformin,
    CASE
        WHEN h.last_hba1c_date IS NULL
          OR DATEDIFF(day, h.last_hba1c_date, CURRENT_DATE) > 180
        THEN 'OVERDUE — HbA1c > 6 months'
        WHEN h.last_hba1c_value > 9.0
        THEN 'POOR CONTROL — HbA1c > 9.0'
        WHEN rx.last_rx_date IS NULL
          OR DATEDIFF(day, rx.last_rx_date, CURRENT_DATE) > 90
        THEN 'MEDICATION GAP — no metformin fill > 90 days'
        ELSE 'COMPLIANT'
    END AS care_gap_flag
FROM diabetic_patients dp
JOIN patient_risk_scores rs ON dp.patient_sk = rs.patient_sk
LEFT JOIN last_hba1c h   ON dp.patient_sk = h.patient_sk
LEFT JOIN last_rx   rx   ON dp.patient_sk = rx.patient_sk
WHERE rs.risk_tier = 'HIGH'
ORDER BY rs.risk_score DESC;

3. Average length-of-stay by diagnosis category (percentile analysis)

SELECT
    dd.category AS diagnosis_category,
    COUNT(*) AS encounter_count,
    ROUND(AVG(fe.length_of_stay_days), 2) AS avg_los,
    PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY fe.length_of_stay_days)
        AS median_los,
    PERCENTILE_CONT(0.9) WITHIN GROUP (ORDER BY fe.length_of_stay_days)
        AS p90_los,
    ROUND(SUM(fe.readmission_flag::INT)::FLOAT / COUNT(*) * 100, 2)
        AS readmission_rate_pct,
    RANK() OVER (ORDER BY AVG(fe.length_of_stay_days) DESC)
        AS rank_by_avg_los
FROM fact_encounters fe
JOIN dim_diagnosis dd ON fe.diagnosis_sk = dd.diagnosis_sk
WHERE fe.discharge_date IS NOT NULL
GROUP BY dd.category
HAVING COUNT(*) >= 100  -- statistical significance filter
ORDER BY avg_los DESC;

Parsing nested FHIR JSON

# FHIR resources have deeply nested arrays — flatten with explode + getItem
from pyspark.sql.functions import explode, col, get_json_object
from pyspark.sql.types import ArrayType, StructType, StructField, StringType

# Schema for FHIR Observation (lab result) resource
obs_schema = StructType([
    StructField("resourceType", StringType()),
    StructField("id", StringType()),
    StructField("status", StringType()),
    StructField("subject", StructType([
        StructField("reference", StringType())
    ])),
    StructField("code", StructType([
        StructField("coding", ArrayType(StructType([
            StructField("system", StringType()),
            StructField("code",   StringType()),
            StructField("display",StringType())
        ])))
    ])),
    StructField("valueQuantity", StructType([
        StructField("value", StringType()),
        StructField("unit",  StringType())
    ]))
])

df_fhir = spark.read.schema(obs_schema).json(bronze_path)

# Flatten: explode coding array, select nested fields
df_flat = (df_fhir
    .withColumn("coding", explode(col("code.coding")))
    .select(
        col("id").alias("observation_id"),
        col("subject.reference").alias("patient_ref"),
        col("coding.code").alias("loinc_code"),
        col("coding.display").alias("test_name"),
        col("valueQuantity.value").cast("double").alias("result_value"),
        col("valueQuantity.unit").alias("unit"),
        col("status")
    )
    .withColumn("patient_id",
        F.regexp_extract(col("patient_ref"), r"Patient/(.+)", 1))
    .drop("patient_ref")
)

Window function: detect 30-day readmissions in PySpark

from pyspark.sql.window import Window

w = Window.partitionBy("patient_id").orderBy("admit_date")

df_readmit = (silver_encounters
    .withColumn("prev_discharge",
        F.lag("discharge_date").over(w))
    .withColumn("days_since_last_discharge",
        F.datediff(col("admit_date"), col("prev_discharge")))
    .withColumn("is_30day_readmission",
        F.when(col("days_since_last_discharge") <= 30, True)
         .otherwise(False))
)

Performance tips specific to healthcare data

  • FHIR JSON is deeply nested — define explicit schemas, never use schema inference on production data
  • Lab result tables can be 100x larger than encounter tables — filter by date partition first before any join
  • Use broadcast(dim_diagnosis) — ICD-10 reference is ~95K rows, perfect for broadcast
  • Patient data skews heavily (ICU patients have 100x more records than routine patients) — use salting technique if joins on patient_id cause data skew
  • Always repartition before writing: .repartition(col("year"), col("month"))
