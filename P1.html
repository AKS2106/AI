Advanced RAG with Hybrid Search, Reranking & Evaluation
Core Skill: Advanced RAG Simulates: Microsoft Copilot / Notion AI
Business Problem & Architecture
▾
Business Problem
Basic RAG fails on 15–20% of real queries — ambiguous questions, typos, domain jargon. No mechanism to detect when answers are hallucinated. No way to measure improvement over time.
Real-World Use Case
Production RAG systems at Microsoft and Notion use hybrid retrieval + reranking to handle complex queries. RAGAS evaluation runs automatically on every deployment as a quality gate.
Architecture Flow (Advanced)
User query → HyDE (GPT generates hypothetical answer) → Embed hypothetical → Dense retrieval from Pinecone (top-20) in PARALLEL with BM25 sparse retrieval (top-20) → Reciprocal Rank Fusion merges lists → Cross-encoder reranker scores all 40 → keeps top-3 → GPT-4o-mini generates answer from top-3 chunks → RAGAS auto-evaluates: faithfulness + answer relevancy
Tech Stack & Folder Structure
▾
Tech Stack
pinecone (free tier)rank-bm25 sentence-transformers cross-encoder langchain EnsembleRetriever ragasopenai gpt-4o-mini fastapistreamlit
Key Concepts Implemented
HyDE: Generate hypothetical answer → better embedding match
BM25: Keyword-based sparse retrieval
RRF: Merge dense + sparse result lists
Cross-encoder: Rerank by query-chunk relevance
Folder Structure
advanced-rag-system/ ├── app/ │ ├── main.py │ ├── retrieval/ │ │ ├── dense.py ← Pinecone │ │ ├── sparse.py ← BM25 │ │ ├── hybrid.py ← RRF fusion │ │ └── reranker.py ← Cross-encoder │ ├── hyde.py ← HyDE logic │ └── rag_chain.py ├── evaluation/ │ ├── eval_dataset.json ← 50 Q&A pairs │ └── ragas_eval.py ├── ui/ │ ├── chat_app.py │ └── eval_dashboard.py └── README.md
Implementation Steps & Interview Prep
▾
1
Set up Pinecone index (free tier)
Create index with dimension=384 (all-MiniLM-L6-v2 output size). Upsert all document chunks with metadata. Verify with describe_index_stats().
2
Implement HyDE in hyde.py
Prompt GPT-4o-mini: "Write a detailed paragraph answering: {query}". Embed the generated answer instead of the original query. This dramatically improves retrieval for vague questions.
3
Build BM25 retriever in sparse.py
Use rank_bm25 library. Index all document chunks. For each query, return top-20 chunks by BM25 score. Store both retrieved lists with their original rank positions.
4
Implement RRF fusion in hybrid.py
RRF score = sum(1/(k + rank_i)) for each document across both lists (k=60). Merge and sort by combined RRF score. Top-40 go to reranker.
5
Add cross-encoder reranking
Use sentence-transformers cross-encoder/ms-marco-MiniLM-L-6-v2 (free, local). Score all 40 (query, chunk) pairs. Keep top-3 highest-scored chunks for generation.
6
Build RAGAS evaluation pipeline
Create eval_dataset.json with 50 Q&A pairs. Run ragas.evaluate() with metrics=[faithfulness, answer_relevancy, context_recall, context_precision]. Log scores per run to track improvement.

When would you use a cross-encoder vs a bi-encoder?

Implementation Steps & Interview Prep
▾
1
Define TypedDict state in state.py
All workflow data lives in one state dict. Nodes receive state, perform their task, return updated state dict. Never mutate state — always return a new dict.
2
Build StateGraph in graph.py
workflow = StateGraph(WorkflowState) → add_node for each of 7 nodes → add_edge for linear transitions → add_conditional_edges at classifier node for issue_type routing.
3
Implement conditional edge routing in edges.py
route_by_issue_type(state) → returns "billing_node" | "technical_node" | "human_review". LangGraph calls this function after classifier node to determine next node.
4
Add human-in-loop with interrupt_before
graph.compile(interrupt_before=["human_review"]). When complex case hits this node, workflow pauses. Ops team reviews via dashboard, approves/modifies, then resumes with graph.invoke(config, input).
5
Stream progress via WebSockets
FastAPI WebSocket endpoint streams node completion events to Streamlit ops dashboard in real-time. Each node emits {node_name, status, timestamp} as it completes.
6
Add LangSmith tracing
Set LANGCHAIN_TRACING_V2=true in .env. Every node execution is automatically traced. View full workflow execution tree, input/output per node, latency, and token cost in LangSmith dashboard.
